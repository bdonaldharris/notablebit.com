# NotableBIT.com Complete Redesign Plan

**Document purpose:** This redesign plan is intended to serve as a source-of-truth reference for drafting Google Stitch design prompts, Codex issue prompts, implementation tickets, and design handoff bundles for the complete rebuild of `notablebit.com`.

**Site:** `https://notablebit.com/`  
**Recommended rebuild approach:** Build from scratch as a modern, focused, static marketing website. Harvest useful content, assets, and domain knowledge from the existing repo, but do not treat the current site as the structural foundation.

---

## 1. Strategic Summary

NotableBIT.com should become the company and studio website for the broader ecosystem B Donald Harris is building.

The current NotableBIT site reflects an earlier version of the work: general consulting, technology services, an older podcast identity, and a broader small-business technology support posture. The redesigned site should shift from “generic tech consulting company” to “founder-led technology studio building products, platforms, media, and strategic systems for the AI era.”

The site should not try to explain everything about B Donald personally. That role belongs to `bdonaldharris.com`. NotableBIT.com should explain what the company does, what it builds, what it offers, and how partners, clients, sponsors, and collaborators can engage.

### Core positioning

> **NotableBIT is a founder-led technology studio building products, platforms, and strategic systems for the AI era.**

### Expanded positioning

> NotableBIT builds at the intersection of software, media, community, and strategic technology — with a focus on Black builders, founder-led products, practical AI adoption, and systems that turn ideas into durable execution.

## 2. Brand Architecture

The redesigned site should clearly position NotableBIT within the larger ecosystem.

### Ecosystem map

| Brand / Site | Role | Purpose |
|---|---|---|
| `bdonaldharris.com` | Personal authority site | B Donald Harris as founder, speaker, builder, technologist, ecosystem architect, and public voice. |
| `notablebit.com` | Company / studio site | The business entity behind products, platforms, strategic services, media, and ecosystem-building work. |
| `bitvoices.network` | Community / media platform | A platform and media network amplifying Black excellence in tech and creating visibility for Black builders. |
| HindSite | Product | Workflow intelligence for builders; captures development traces and turns them into timelines, reflections, and publishable artifacts. |

### NotableBIT’s specific job

NotableBIT.com should answer:

> **What is the company behind this work, what does it build, and how can people work with it?**

### What NotableBIT should not be

NotableBIT should not read like:

- a generic IT support company
- a broad digital transformation agency
- a personal ministry or coaching website
- a podcast-only brand
- a scattered list of unrelated ventures
- a site promising future capabilities that are not yet active

### What NotableBIT should become

NotableBIT should read like:

- a product and technology studio
- a strategic AI-era builder partner
- the company home for HindSite and BitVoices
- a vehicle for consulting, product strategy, media, and community initiatives
- a founder-led business with a clear point of view

## 3. Primary Audience

The site must speak to multiple audiences without becoming unfocused.

### Primary audiences

1. **Founders and builders** who need product strategy, specs, MVP planning, or AI workflow guidance.
2. **Small to mid-sized businesses** that need strategic technology leadership, not generic tech support.
3. **Partners and sponsors** interested in Black tech ecosystem-building, community, events, or media collaboration.
4. **Product users and early adopters** discovering HindSite, BitVoices, or future NotableBIT-built products.
5. **Event organizers and ecosystem leaders** who want to understand the company behind B Donald’s public work.

### Audience mindset

Visitors may be asking:

- What does NotableBIT actually do now?
- Is this a consulting company, product company, media company, or community company?
- How does NotableBIT relate to BitVoices and HindSite?
- Can I hire NotableBIT?
- Can I partner with NotableBIT?
- What makes this company different from a generic agency?

The redesign should answer these questions quickly.

## 16. Codex Implementation Guidance

The rebuilt site should be simple, maintainable, and content-driven.

### Recommended implementation approach

- Build from scratch.
- Use React or Next.js depending on preferred deployment setup.
- Keep the site static unless dynamic features are required.
- Use reusable components.
- Keep copy in structured content files where practical.
- Avoid adding a CMS at this stage unless there is a clear publishing workflow.

### Suggested technical stack

Option A:

- React
- Vite
- TypeScript
- Tailwind CSS
- React Router if needed

Option B:

- Next.js
- TypeScript
- Tailwind CSS
- Static export or standard Vercel deployment

### Recommended component structure

```txt
src/
  components/
    layout/
      Header.tsx
      Footer.tsx
      PageShell.tsx
    sections/
      HeroSection.tsx
      EcosystemSection.tsx
      ServicesSection.tsx
      ProductFeatureSection.tsx
      MediaSection.tsx
      FounderSection.tsx
      CtaSection.tsx
    ui/
      Button.tsx
      Card.tsx
      Badge.tsx
      SectionHeading.tsx
      LinkCard.tsx
  content/
    site.ts
    products.ts
    services.ts
    media.ts
  pages/
    Home.tsx
    Studio.tsx
    Products.tsx
    Consulting.tsx
    Media.tsx
    About.tsx
    Contact.tsx
```

### Routing

Recommended routes:

```txt
/
/studio
/products
/consulting
/media
/about
/contact
```

### Redirects

Add redirects from old pages where applicable.

Possible old routes to redirect:

```txt
/services -> /consulting
/podcast -> /media
/swag -> /
/tech-education -> /consulting or /studio
/contact -> /contact
```

Exact redirects should be confirmed by reviewing existing routes before implementation.

### SEO basics

Each page should include:

- unique title
- meta description
- Open Graph title
- Open Graph description
- canonical URL
- social preview image if available

### Accessibility requirements

- Semantic HTML
- Keyboard accessible navigation
- Good color contrast
- Clear focus states
- Proper heading hierarchy
- Descriptive links
- Alt text for meaningful images
- Avoid motion-heavy experiences without reduced-motion support

### Performance requirements

- Static-first pages
- Optimized images
- Minimal animation libraries
- No unnecessary client-side JavaScript
- Fast first load
- Responsive layouts
